# 開発ルール（TAITAI-TECH）

1. **スタック方針**
   - Next.js App Router + TypeScript を利用し、サーバーアクションなど Vercel 依存機能は当面使用しない。
   - UI は Tailwind CSS を基本とし、カスタムスタイルは `globals.css` に変数として定義した上で利用する。

2. **ディレクトリ構成**
   - `src/app` 配下にページを配置。共通コンポーネントは `src/components` にまとめ、用途別サブディレクトリを作成する。
   - 各セクションの JSX は可能な限りコンポーネント化して再利用性を担保する。

3. **スタイルとアクセシビリティ**
   - カラーは `--color-primary (#B3424A)` を軸にアクセシビリティを意識して使用する。
   - インタラクティブ要素には `aria-*` 属性や `aria-live` など適切なアクセシビリティ対応を行う。

4. **コンテンツ運用**
   - 社外公開向け文章は日本語で統一する。
   - タイポグラフィや CTA のトーンは「学生が運営するプロフェッショナル集団」であることを意識する。

5. **問い合わせフォーム**
   - 現時点ではダミー送信関数を利用し、サーバー連携が必要になった段階で API レイヤーを実装する。
   - エラーメッセージはユーザーにわかりやすくフィードバックする。

6. **開発フロー**
   - 大きめの変更時は TODO リストを用いて着手状況を管理する。
   - Lint/format はコミット前に実行し、必要に応じて `npm run lint` を利用する。

7. **デプロイ方針**
   - Vercel 以外のホスティングも想定し、環境変数やビルドコマンドは標準的な Next.js のみで完結させる。

このルールファイルは必要に応じてアップデートしてください。
